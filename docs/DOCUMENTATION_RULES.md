# Documentation Rules

Правила для любого помощника, который работает с этим репозиторием.

## 1) Когда документация обязательна

Обновляйте документацию в каждом PR, если изменилось хотя бы одно:
- новый/изменённый secret;
- новый/изменённый input workflow;
- новый feature flag;
- новый параметр роли;
- изменение поведения режима `bootstrap/deploy/lockdown`;
- изменение контракта fleet-конфига.

## 2) Что обновлять

Минимум:
- `README.md` — короткая версия для быстрого входа.
- `docs/OPERATIONS_GUIDE.md` — пошаговая инструкция для оператора.
- `docs/ROLE_CATALOG.md` — каталог ролей, флагов, зависимостей.

## 3) Стиль документации

- Пишите простыми фразами, без жаргона где это возможно.
- Для каждого нового параметра указывайте:
  - что это,
  - где задаётся,
  - default,
  - что сломается при неправильном значении.
- Всегда добавляйте рабочий пример (YAML/JSON/CLI).
- Для критичных операций добавляйте раздел `Проверка результата`.

## 4) Шаблон для новых переменных

Используйте такой формат:

```md
### `variable_name`
- Где задаётся: `<секрет / workflow input / group_vars / fleet config>`
- Тип: `<bool|str|int|list>`
- Default: `<значение>`
- Обязательность: `<обязателен/опционален и при каких условиях>`
- Пример:
  ```yaml
  variable_name: value
  ```
```

## 5) Шаблон для новых ролей

```md
## `role_name`
- Назначение: ...
- Когда запускается: ...
- Ключевые переменные: ...
- Риски: ...
- Пример включения во fleet config: ...
```

## 6) Проверки перед merge

Перед merge помощник обязан выполнить:

```bash
python .github/scripts/test-render-fleet-runtime.py
ansible-playbook -i hosts.example.ini playbook.yml --syntax-check
ansible-lint playbook.yml roles
yamllint .
```

Если проверки не прошли, в PR нужно явно указать:
- какая команда упала;
- почему;
- какой план исправления.

## 7) Чего делать нельзя

- Нельзя добавлять реальные пароли/ключи в репозиторий.
- Нельзя оставлять изменения контракта без обновления примеров.
- Нельзя менять режимы workflow без обновления runbook.
- Нельзя удалять старые инструкции, пока не добавлены новые эквивалентные шаги.
